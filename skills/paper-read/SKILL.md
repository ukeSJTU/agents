# 论文三遍研读

将论文阅读为可回溯的研读记录，不把流畅的概述当作证据。固定完成
First Pass、Second Pass 和 Third Pass；阅读深度决定每一遍的展开程度，
不改变三遍结构。

## 开始前

1. 确认输入是用户提供的论文 PDF，以及用户是否同时提供附录、补充材料或
   指定阅读范围。
2. 确认用户明确指定阅读深度。只接受 `理解`、`精读` 或 `审读`。若未指定，
   在开始阅读前仅追问这一项，不自行选择。
3. 除非用户明确要求，不检索互联网、不读取 PDF 以外的资料，也不补充背景
   知识。用户额外提供或明确要求查找的资料按 `X` 类来源处理。
4. 默认在回复中交付 Markdown；只有用户指定路径或明确要求保存时才创建
   文件。文件名使用 `{PDF 文件名或论文标题}_研读记录.md`。

在此仓库中，优先结合 `$pdf-read` 读取 PDF：先检查文件和原生文本，再按需
提取带页码的文本块、表格或渲染页面。不要复制它的脚本或绕过它的运行限制。
如果 `$pdf-read` 不可用，使用可用的 PDF 阅读能力，并维持同等的页级证据
标准。

## 阅读深度

三种深度都必须完成三遍。不要用“理解”替代 First Pass，也不要因为“审读”
跳过方法和结果的准确复述。

| 深度   | First Pass                                       | Second Pass                                  | Third Pass                                                               |
| ------ | ------------------------------------------------ | -------------------------------------------- | ------------------------------------------------------------------------ |
| `理解` | 建立全文地图、问题、方法和主要结果的轮廓。       | 解释核心机制与最重要证据；记录一个关键论证。 | 核验最关键主张，标出明显的适用条件和未知项。                             |
| `精读` | 完整建立论文地图，标出关键图表、公式和附录入口。 | 拆解方法、实验和一至三条关键论证。           | 分别评估核心主张、证据边界、限制和替代解释。                             |
| `审读` | 在完整地图上标出可能影响结论的结构、定义与范围。 | 精确记录支撑主要结论的实验、推导和比较。     | 逐项核验摘要或结论中的主要主张；检查隐含假设、遗漏条件、限制和替代解释。 |

## 论文主线｜一遍读懂

把“论文主线｜一遍读懂”作为最终记录的第一个正文部分。它是根据三遍阅读写成的
叙事综合，不是 First Pass 之前的预设结论，也不是第四遍阅读。

1. 在 **First Pass** 建立内部故事骨架：领域处境与缺口 → 研究问题 → 作者的
   回应 → 需要检验的结果。不要在此时交付叙事。
2. 在 **Second Pass** 用方法、实验、推导或文献比较补全骨架，并为每一环建立
   `E`。
3. 在 **Third Pass** 检查结论的成立条件、限制、替代解释和待核验处。用这些
   内容修正故事的结尾。
4. 三遍完成后才写连续叙事，并将它放在标题之后、First Pass 之前。

按因果与论证关系写，不按论文章节顺序复述。叙事必须依次让读者理解：原有
处境与真正缺口、作者的问题与成功标准、作者为何选择这个方案、如何检验、
最有决定性的发现，以及结论在何种条件下成立。结尾必须保留最重要的限制、
不确定性或未解决问题，不能以“作者成功了”收尾。

使用连续段落，不使用项目符号、编号或段内小标题。每个事实性句子仍引用
`E`；归纳、范围判断和批判标为 `I`；无法补全的叙事连接处标为 `Q`。不要为
了让故事完整而引入 PDF 外背景、未报告的动机或选择性忽略不利结果。

| 深度   | 论文主线的展开                                           |
| ------ | -------------------------------------------------------- |
| `理解` | 用一至两段交代完整主线和一个最重要的结论边界。           |
| `精读` | 用三至四段说明方案选择、决定性证据和结论条件。           |
| `审读` | 用四至六段额外说明最可能改变判断的假设、反例或证据缺口。 |

## 阅读工作流

### 1. 建立可检索的论文地图

1. 检查 PDF 的页数、可提取文本、目录和版式异常。
2. 阅读主文全部章节以建立章节—页码地图；不可只根据摘要、引言和结论写
   全文笔记。
3. 定位摘要、方法、结果、结论、图表、公式、附录和补充材料。对承载关键
   结论的图表、公式、复杂表格或多栏文本渲染原页核对。
4. 对未可靠解析的页面或视觉内容保留异常记录。不要用未核验的提取结果补全
   内容，也不要把未读附录写成“已检查”。

### 2. 按三遍形成笔记

先读取 [references/evidence-policy.md](references/evidence-policy.md)，再读取
[references/note-schema.md](references/note-schema.md)。按以下顺序工作：

1. **First Pass**：只建立研究问题、作者声称的答案、论文结构和回读线索；
   不提前接受或反驳结论。
2. **Second Pass**：记录作者如何从动机走到方法、实验、结果和关键论证；
   先建立或复用 `E`，再写解释。
3. **Third Pass**：区分作者报告的内容和自己的判断，检查证据是否足以支持
   主张及其外推范围。不要加入环境配置或实现推测。
4. **叙事综合**：完成三遍后，按“论文主线｜一遍读懂”的规则组装连续叙事；
   它展示在记录最前，但必须最后生成。

按用户选择的深度调整细节。理论论文可省略不适用的实验字段；经验论文可
省略不适用的公式字段；不要留下空白模板项。

## 证据与交付规则

1. 使用 `E`、`I`、`Q`、`X` 短索引，并在末尾提供证据索引。严格遵守
   `references/evidence-policy.md`。
2. 给每一项关键本文事实、数值、方法设置、限制和作者结论提供至少一个 `E`。
   将归纳、因果解释、批判或跨证据比较标为 `I`，并指回其依据。
3. 对图表、公式和表格相关结论，`E` 必须标明对应编号和页码；无法可靠读取
   时建立 `Q`，不要猜测。
4. 标题使用 `# {论文标题}｜研读记录`；标题不可确定时使用 PDF 文件名。标题
   后直接进入“论文主线｜一遍读懂”，不要插入阅读深度、范围或依据的元数据块。
5. 使用中文 Markdown，除非用户另有要求。以完整研读记录为主要交付物；
   回复末尾可用一句话说明提取异常或未阅读的材料，但不要用它替代记录。

## 交付前检查

- 是否按用户指定的阅读深度完成全部三遍？
- 论文主线是否最后生成、展示在记录最前，并用连续叙事覆盖完整论证链及其边界？
- 每个关键事实是否能经 `E` 的检索锚点与章节、页码、图表或公式回到原文？
- 每个 `I` 是否与作者陈述清楚分开，并列出 `E` 或 `X` 依据？
- `Q` 是否说明要核验什么、应查看哪里，而非笼统地写“不确定”？
- 是否避免了 PDF 外事实、未核验的视觉解释、实现建议和空白字段？
- 是否如实说明解析异常，以及附录或补充材料的实际阅读范围？
