# 研读记录结构

使用以下结构生成完整记录。根据论文类型与阅读深度裁剪条目：删除不适用的字段，
不要保留空行、空标题或空白占位符。事实性内容必须按
[证据与定位规范](evidence-policy.md) 标注索引。

```markdown
# {论文标题}｜研读记录

## 论文主线｜一遍读懂

{用连续叙事串联：原有处境与缺口 → 研究问题 → 作者方案及其理由 → 检验方式
→ 决定性发现 → 有条件的结论与关键边界。不要分点；关键句使用〔E..〕、〔I..〕
或〔Q..〕。}

## First Pass｜建立论文地图

### 论文概要

- 标题、作者与发表信息：
- 研究领域与关键词：
- 一句话概述：

### 论文地图

- 论文要解决的问题与其重要性：
- 作者提出的核心答案、方法或系统：
- 作者声称的主要贡献：
- 主要证据：
- 作者明确说明的限制：
- 最相关的已有工作：
- 关键图、表、公式或示意图：

### 建议回读

- {需要在后两遍确认的具体问题}

---

## Second Pass｜拆解方法与证据

### 研究问题与动机

- 具体研究问题或假设：
- 现有方法的不足：
- 作者的出发点：

### 方法与技术细节

- 核心想法：
- 方法或系统流程：输入 → 关键步骤 → 输出：
- 关键模块、算法、设计选择或推导：
- 关键术语、公式或概念：
- 与已有方法的区别：

### 实验与结果

- 任务、数据集或研究场景：
- 实验或分析设置：
- 对比方法与评价指标：
- 核心结果与关键数值：
- 关键图表及其含义：
- 作者据此得出的结论：

### 关键论证

#### 论证 {N}

- 作者主张：
- 支撑证据：
- 适用范围与条件：
- 作者未覆盖或仅简略讨论的部分：

### 关联与批注

- 重要引用与后续待读：
- 值得保留的论点、证据或表述：
- 疑问、歧义或需要核对之处：
- 一句话总结：

---

## Third Pass｜核验主张与边界

### 核心主张核验

#### 主张 {N}

- 主张：
- 原文证据与位置：
- 隐含假设：
- 相关限制、失败情况或未覆盖条件：
- 可能的反例或替代解释：
- 是否足以支持主张：

### 使用范围与风险

> 仅在模型、数据集、应用系统或风险确实相关时保留。

- 作者声明的预期使用场景与对象：
- 明确不适用、未验证或可能被误用的场景：
- 可靠性、安全、隐私、公平性、环境或其他风险：
- 作者提出的缓解措施、披露或未解决问题：

### 研究脉络与延伸

- 所属研究线及与相邻工作的关键差异：
- 仍缺少的证据或未解决的问题：
- 可借鉴到当前工作的部分：
- 最终研读判断：

---

## 证据索引

### E｜原文证据

- E01｜{短标题}｜检索锚点：`{原文短语}`｜{章节 · 页码 · 图表/公式}

### I｜研读解释

- I01｜{短标题}｜基于 E01、E02

### Q｜待核验

- Q01｜{具体问题}｜检查范围：{章节、页码或 E 条目}

### X｜外部资料

- X01｜{来源链接或名称}｜核对日期：{YYYY-MM-DD}｜{仅在实际使用外部资料时保留}
```

## 深度下的最小覆盖

| 深度   | 论文主线                                 | Second Pass 的覆盖                               | Third Pass 的覆盖                          |
| ------ | ---------------------------------------- | ------------------------------------------------ | ------------------------------------------ |
| `理解` | 一至两段，完整主线加一个重要边界。       | 方法或论证择最核心部分说明；覆盖主要结果。       | 核验一个最关键主张。                       |
| `精读` | 三至四段，说明方案、证据和结论条件。     | 覆盖论文核心方法、主要设置、结果与一至三条论证。 | 分别核验核心主张和主要限制。               |
| `审读` | 四至六段，加入关键假设、反例或证据缺口。 | 覆盖所有支撑主要结论的关键实验、推导或比较。     | 对摘要或结论中的每项主要主张建立核验条目。 |
