# 证据与定位规范

先区分来源与判断，再写笔记。正文使用短索引，完整依据放在末尾的“证据索引”。

```text
模型在任务 X 上优于基线。〔E07〕
该改动可能主要改善了长尾样本。〔I03〕
作者是否控制了计算预算仍不清楚。〔Q02〕
代码仓库未列出随机种子。〔X01〕
```

一个条目只写一个可核验的主张。一个主张有多个依据时可引用多个索引，例如
`〔E03、E07〕`。

## 来源边界

1. 叙述“本文”时，只依据用户提供的论文 PDF、附录和补充材料。
2. 不将相关工作、一般背景、代码仓库、数据集页或网络资料混入本文结论。
   它们必须是 `X`。
3. 不补全原文没有报告的实验、数值、比较、限制或引用。证据不足时建立 `Q`。
4. 人的原文阅读和判断优先于本记录；本记录不是正式引用的替代品。

## 索引类型

### E｜原文证据

`E` 只记录作者明确写出、报告或展示的内容：研究问题、方法、实验设置、
数字结果、作者声明的限制和相关工作。

每项使用以下形式：

```text
E07｜模型结果｜检索锚点：`GPT-5 19.2`｜§4.2 · p.6 · Table 1
E08｜核心方法｜检索锚点：`two-stage training`｜§3.2 · p.5 · Fig.2
```

- 检索锚点选原文中可搜索的 2–12 个词或短语；图、表、公式可使用编号。
- 定位尽可能含章节、当前 PDF 版本页码、图、表、公式、附录或补充材料。
- 不把无关事实塞入同一项；同一段或图表可支撑多个正文条目。

### I｜研读解释

`I` 记录基于原文的归纳、比较、因果解释、批判或范围判断。不能把 `I` 写成
作者结论，且必须列出依据：

```text
I03｜该设计可能主要改善长尾样本｜基于 E07、E09
```

### Q｜待核验

`Q` 记录证据不足、表述含糊、解析不可靠，或必须回看图表、公式、附录的内容。
说明具体问题和检查范围：

```text
Q02｜是否控制计算预算｜检查范围：E10、Appendix B
Q03｜是否报告统计显著性｜检查范围：§4.1–§4.5；定位：未找到
```

### X｜外部资料

`X` 仅用于用户提供或明确要求查找的 PDF 外资料。给出链接或明确来源及核对
日期：

```text
X01｜[GitHub README](https://example.com)｜核对日期：2026-07-14｜未列出随机种子
```

基于外部资料的判断仍使用 `I`，并同时引用相应 `X`。用户未要求外部资料时，
不要创建 `X` 条目。

## 图表、公式与解析异常

1. 图表结论同时记录图表编号、图注或相关正文位置。
2. 公式解释记录公式编号及变量定义位置。
3. 代码或实现相关内容只记录论文明确说明的部分；不得从实现反推为论文事实。
4. PDF 解析、视觉检查或版式阅读不可靠时，建立 `Q` 并说明受影响的页和区域。
   不以流畅措辞掩盖缺失内容。
